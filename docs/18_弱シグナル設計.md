# Hospital CRM 弱シグナル設計

## 1. 目的

弱シグナル設計は、病院営業・地域連携において、まだ明確な紹介や依頼にはなっていないが、将来的な営業機会やリスクにつながる小さな変化を記録・分析することを目的とする。

## 2. 弱シグナルとは

弱シグナルとは、現時点では小さな情報であっても、将来的に重要な意思決定材料になる可能性がある情報を指す。

例:

- MSWの異動
- 医師の異動
- 地域連携室の名称変更
- 退院支援看護師の増減
- 地域包括ケア病床の増減
- 法人内サービスの新設
- 法人内居宅介護支援事業所の閉鎖
- 紹介件数の増減
- 面談時の反応の変化
- 訪問看護への関心
- 医療依存度の高いケースへの関心

## 3. 弱シグナルの入力元

- 営業履歴
- 訪問記録
- 担当者メモ
- ImportSessionの差分
- PDF更新差分
- 法人サービス情報
- 紹介実績の変化

## 4. データ構造案

候補フィールド:

- priority
- is_resolved
- memo

## 5. AI活用方針

AIは弱シグナルの判断者ではなく、抽出・整理・説明の補助役とする。

AIが支援すること:

- 営業履歴から弱シグナル候補を抽出
- ImportSession差分から変化点を抽出
- 次回訪問時の確認事項を提案
- 営業優先度の理由を説明
- 未確認情報を確認事項として提示

AIが行わないこと:

- 最終判断
- 医療判断
- 介護方針決定
- 未確認情報の断定
- 利用者個人情報の処理

## 6. 営業への活用

弱シグナルは以下に活用する。

- 次回訪問提案
- 営業優先度再評価
- 長期未訪問病院の再訪問理由
- 法人変化の把握
- 地域連携上の変化把握
