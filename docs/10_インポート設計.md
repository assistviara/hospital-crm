# 1. 目的

本ドキュメントは、Hospital CRM におけるデータインポート機能の設計方針を定義する。

病院情報・法人情報などを外部データから効率的に取り込み、手入力を最小限にすることを目的とする。

---

# 2. 基本方針

インポート機能では以下を重視する。

- 手入力を減らす
- 元データを保持する
- インポート履歴を残す
- 同一データは更新する
- AIによる補完を可能にする

---

# 3. 対応ファイル

Version0.1では以下に対応する。

|形式|用途|
|---|---|
|PDF|病院一覧・相談窓口一覧|
|Excel (.xlsx)|病院マスタ|
|CSV|病院・法人情報|
|UTF-8 CSV|一括更新|

将来的には

- JSON
- API
- オープンデータ

にも対応する。

# 4. インポート対象

## 病院マスタ

- 病院名
- 法人名
- 住所
- 電話番号
- FAX番号
- 病床数
- 地域包括ケア病床
- 回復期リハ病床
- 療養病床
- 精神病床
- MSW数
- 退院支援看護師数

---

## 法人マスタ

- 法人名
- 法人種別
- 居宅介護支援
- 訪問看護
- 老健
- サ高住
- 有料老人ホーム

# 5. インポートフロー

```text
ファイル選択

↓

アップロード

↓

フォーマット判定

↓

データ読込

↓

バリデーション

↓

重複判定

↓

登録・更新

↓

結果表示
```

---

# 6. 重複判定

重複判定は以下を基本とする。

## 病院

優先順位

1. 病院名＋住所
2. 病院名
3. 電話番号

一致した場合

- 更新
- スキップ
- 手動選択

を選択できるようにする。

---

## 法人

優先順位

1. 法人名
2. ホームページURL

# 11. ImportService

インポート処理は

```text
apps/importer/services.py
```

へ実装する。

例

```python
ImportService

import_pdf()

import_excel()

validate_data()

merge_duplicate()

save_database()
```

# 12. インポートログ

インポート実行時は履歴を保存する。

管理項目

- 実行日時
- 実行者
- ファイル名
- 登録件数
- 更新件数
- エラー件数

将来的には

ImportHistory

モデルを追加する。

---

# 13. エラー処理

エラーがあった場合

- 登録件数
- 更新件数
- エラー件数

を表示する。

エラー内容も一覧表示する。

例

```text
5行目

病院名がありません
```

# 14. AI活用

AIは以下を支援する。

- PDFレイアウト解析
- 法人名補完
- 病院名補正
- 表記ゆれ補正
- 住所補完
- 不足データ候補提示

AIは自動登録を行わず、

候補を提示するだけ

とする。

---

# 15. 設計上の注意点

- 元ファイルは変更しない
- インポート前にバックアップする
- 同一データは重複登録しない
- エラーは途中停止ではなく可能な限り継続する
- AI補完結果は人が確認して登録する
