# 1. 目的

本ドキュメントは、Hospital CRM におけるAI機能の設計方針を定義する。

AIは営業担当者の判断を代替するものではなく、病院情報・法人情報・営業履歴・紹介実績をもとに、要約・説明・提案を行う判断補助機能として利用する。

---

# 2. AI利用の基本方針

AIは以下の方針で利用する。

- 最終判断は人が行う
- AIは営業判断の補助に限定する
- 利用者個人情報はAIに送信しない
- AI出力は保存前に人が確認する
- AIの回答は根拠データと一緒に表示する
- 営業順位の決定はAnalyticsServiceが行う
- AIは営業順位の理由説明を補助する

---

# 3. AIの役割

AIは以下の役割を担う。

|役割|内容|
|---|---|
|要約|営業履歴・面談記録を短く整理する|
|説明|営業スコアの理由を分かりやすく説明する|
|提案|次回訪問時の話題や確認事項を提案する|
|抽出|好反応・懸念点・弱シグナルを抽出する|
|比較|病院・法人の特徴を比較する|
|文章生成|営業コメント・訪問メモ・報告文を作成する|

---

# 4. AIに入力するデータ

AIに入力できる情報は以下とする。

- 病院名
- 法人名
- 病床構成
- 法人サービス情報
- 地域連携部門情報
- MSW数
- 退院調整看護師数
- 営業履歴要約
- 紹介件数
- 契約件数
- タグ
- 情報信頼度
- フォロー状況

---

# 5. AIに入力しないデータ

以下の情報はAIに入力しない。

- 利用者氏名
- 利用者住所
- 生年月日
- 診断名
- 詳細な病状
- 家族情報
- 個別ケースの具体的経過
- 個人が特定される相談内容
