# 1. 目的

本ドキュメントは、Hospital CRM における病院マスタの設計を定義する。

病院マスタは、本システムの中核となる情報であり、営業履歴・担当者情報・紹介実績・営業分析の基礎データとなる。

---

# 2. 病院マスタの役割

病院マスタでは、以下の情報を管理する。

- 病院基本情報
- 法人との紐づき
- 病床構成
- 地域連携部門
- MSW数
- 退院調整看護師数
- 営業分析に利用する基礎情報

---

# 3. 対象データ

初期データは、久留米医師会等の病院相談窓口一覧PDFをもとに登録する。

主な取込対象は以下とする。

- 病院名
- 全病床数

- MSW数
- 退院調整看護師数

# 4. Djangoモデル対応

病院マスタは `Hospital` モデルで管理する。

```python
class Hospital(BaseModel):
    corporation = models.ForeignKey(
        Corporation,
        on_delete=models.SET_NULL,
        null=True,
        blank=True,
        related_name="hospitals"
    )

    hospital_name = models.CharField(max_length=255)
    address = models.CharField(max_length=255, blank=True)
    phone = models.CharField(max_length=50, blank=True)
    fax = models.CharField(max_length=50, blank=True)
    homepage_url = models.URLField(blank=True)

    total_beds = models.IntegerField(default=0)
    general_beds = models.IntegerField(default=0)
    chronic_beds = models.IntegerField(default=0)
    psychiatric_beds = models.IntegerField(default=0)
    community_beds = models.IntegerField(default=0)
    recovery_beds = models.IntegerField(default=0)
    disability_beds = models.IntegerField(default=0)
    other_beds = models.IntegerField(default=0)

    has_regional_cooperation = models.BooleanField(default=False)
    regional_department_name = models.CharField(max_length=255, blank=True)

    msw_count = models.IntegerField(default=0)
    discharge_nurse_count = models.IntegerField(default=0)
```

# 10. バリデーション

## 10.1 必須項目

- 病院名

## 10.2 数値項目

病床数・MSW数・退院調整看護師数は0以上の整数とする。

## 10.3 病床数チェック

総病床数と各病床数の合計が大きく異なる場合は警告を表示する。

```text
一般 + 療養 + 精神 + 地域包括 + 回復期 + 障害者 + その他 ≒ 総病床数
```

ただし、PDF上で休床や実働病床が記載される場合があるため、エラーではなく警告とする。

# 12. 営業分析との関係

Hospitalモデルの病床情報は、営業分析の基礎データとして利用する。

AnalyticsServiceでは、以下を計算する。

- 在宅復帰指数
- 営業可能性指数
- 総合営業優先度
- 営業ランク

病床構成の考え方は以下とする。

|病床種別|営業分析上の意味|
|---|---|
|地域包括ケア病床|在宅復帰可能性が高い|
|回復期リハ病床|在宅復帰可能性が高い|
|一般病床|退院支援対象が発生しやすい|
|療養病床|長期療養中心で在宅復帰は限定的|
|精神病床|通常の居宅紹介とは別軸で評価|
|障害者病床|個別評価|
|その他病床|個別評価|

# 14. 設計上の注意点

- 病院情報は法人情報と分離する
- 法人が未調査の場合でも病院登録を可能にする
- PDF由来データとWeb調査データを混在させるため、備考欄を活用する
- 病床数の不一致はエラーではなく警告扱いにする
- 営業ランクは保存せず、AnalyticsServiceで算出する
- 利用者個人情報は病院マスタには保持しない
