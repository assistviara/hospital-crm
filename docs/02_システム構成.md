# 1. システム構成概要

Hospital CRM は、Django を用いたWebアプリケーションとして構築する。

初期段階では、事業所単体での利用を想定し、データベースには SQLite を採用する。

将来的に利用者数やデータ量が増加した場合でも、Django ORM を利用することで PostgreSQL などへの移行を可能とする。

# 6. レイヤー構成

本システムでは、処理を以下のレイヤーに分ける。

```text
Template
  ↓
View
  ↓
Service
  ↓
Model
  ↓
Database
```

---

## 6.1 Template層

利用者に表示するHTML画面を担当する。

主な役割：

- 一覧画面
- 詳細画面
- 入力フォーム
- ダッシュボード表示

---

## 6.2 View層

リクエストを受け取り、画面表示やフォーム処理を制御する。

View層には複雑な業務ロジックを書かない。

---

## 6.3 Service層

営業スコア計算、AIコメント生成、インポート処理などの業務ロジックを担当する。

例：

```text
HospitalService
CorporationService
VisitService
AnalyticsService
ImportService
```

---

## 6.4 Model層

データベース構造を定義する。

Django ORM により、SQLiteと連携する。

# 11. 設計上の注意点

- Viewに業務ロジックを集中させない
- スコア計算は analytics の Service層に集約する
- PDF取込処理は importer に分離する
- 病院情報と法人情報は分離する
- 利用者個人情報は扱わない
- 将来的なAI利用を考慮し、データ構造を整える
