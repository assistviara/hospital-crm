# 1. 目的

本ドキュメントは、Hospital CRM における営業履歴の設計を定義する。

営業履歴は、病院・法人・担当者との接点を記録し、営業活動を個人の記憶ではなく、組織の資産として蓄積するための中核データである。

---

# 2. 営業履歴の役割

営業履歴では以下を管理する。

- 訪問日
- 訪問方法
- 訪問者
- 面談相手
- 面談内容
- 好反応だった内容
- 懸念点
- 次回対応
- 次回訪問予定
- 営業メモ

---

# 3. Djangoモデル対応

営業履歴は `VisitRecord` モデルで管理する。

```python
class VisitRecord(BaseModel):
    hospital = models.ForeignKey(
        Hospital,
        on_delete=models.CASCADE,
        related_name="visit_records"
    )

    contact_person = models.ForeignKey(
        ContactPerson,
        on_delete=models.SET_NULL,
        null=True,
        blank=True,
        related_name="visit_records"
    )

    visit_date = models.DateField()
    visit_method = models.CharField(max_length=50, blank=True)
    visitor_name = models.CharField(max_length=100, blank=True)

    summary = models.TextField(blank=True)
    positive_response = models.TextField(blank=True)
    concern = models.TextField(blank=True)
    next_action = models.TextField(blank=True)
    next_visit_date = models.DateField(null=True, blank=True)

    information_level = models.CharField(
        max_length=20,
        choices=[
            ("official", "公式情報"),
            ("confirmed", "確認済み"),
            ("heard", "聞き取り"),
            ("rumor", "未確認"),
        ],
        default="heard"
    )

    tags = models.CharField(max_length=500, blank=True)

    follow_status = models.CharField(
        max_length=20,
        choices=[
            ("todo", "未対応"),
            ("doing", "対応中"),
            ("done", "完了"),
        ],
        default="todo"
    )
```

# 8. 検索・絞り込み

営業履歴一覧では以下の検索条件を用意する。

|検索条件|内容|
|---|---|
|病院名|部分一致検索|
|法人名|部分一致検索|
|訪問者|部分一致検索|
|面談相手|部分一致検索|
|訪問方法|訪問・電話・メール等|
|訪問日範囲|開始日〜終了日|
|次回訪問予定あり|next_visit_date is not null|
|好反応あり|positive_response が空でない|
|懸念点あり|concern が空でない|

# 11. 営業分析との関係

営業履歴は、AnalyticsService において以下の分析に利用する。

- 最終訪問日
- 訪問回数
- 訪問頻度
- 反応が良い話題
- 懸念点の有無
- 次回訪問予定の有無
- 紹介実績との関連

---

# 12. AI活用との関係

営業履歴は将来的にAI分析の入力データとなる。

AIは以下を支援する。

- 面談記録の要約
- 次回訪問提案
- 営業コメント生成
- 好反応だった話題の抽出
- 弱シグナル分析
- 営業優先度の再評価

AI出力は判断補助であり、最終判断は営業担当者が行う。

---

# 13. 入力時の注意点

営業履歴には、利用者個人情報を入力しない。

入力してよい情報：

- 病院側の反応
- 担当者の関心
- 法人サービスに関する情報
- 営業上の課題
- 次回対応

入力しない情報：

- 利用者氏名
- 診断名
- 詳細な病状
- 個別ケースの具体的経過
- 個人が特定される相談内容

# 15. 設計上の注意点

- 営業履歴は病院単位で蓄積する
- 面談相手が不明な場合でも記録できるようにする
- 担当者が退職・異動しても履歴は残す
- 利用者個人情報は入力しない
- AI分析に使いやすいよう、要約・好反応・懸念点を分けて記録する
- 次回対応を明確に残す
- 営業履歴は営業分析の重要な入力データとする
