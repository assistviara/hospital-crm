# 1. 目的

本ドキュメントは、Hospital CRM における画面設計を定義する。

本システムでは、病院情報・法人情報・営業履歴・営業分析を、営業担当者が短時間で確認・入力できる画面構成とする。

---

# 2. 画面設計の基本方針

画面設計では以下を重視する。

- 入力しやすい
- 一覧で比較しやすい
- 営業前に必要情報を素早く確認できる
- 病院詳細画面に情報を集約する
- スコアやAIコメントは補助情報として表示する
- スマートフォン表示にも将来対応できる構成とする

14. 設計上の注意点
病院詳細画面を中心画面とする
営業前に必要な情報を1画面で確認できる構成にする
営業履歴登録は入力項目を多くしすぎない
AIコメントは補助情報として表示する
スコアは保存せず、表示時に計算する
個人情報を表示・入力しない
スマートフォン対応を将来考慮する
