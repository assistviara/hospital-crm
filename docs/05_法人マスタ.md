# 1. 目的

本ドキュメントは、Hospital CRM における法人マスタの設計を定義する。

法人マスタは、病院を運営する医療法人・学校法人・社会医療法人等の情報を管理し、営業分析における競合度・法人内完結度を判断するための基礎データとする。

---

# 2. 法人マスタの役割

法人マスタでは以下を管理する。

- 法人基本情報
- 法人が運営する病院
- 法人内の介護保険サービス
- 法人内居宅介護支援事業所の有無
- 法人内訪問看護の有無
- 老健・サ高住・有料老人ホーム等の有無
- 営業上の競合度
- 法人内完結傾向

---

# 3. Djangoモデル対応

法人マスタは `Corporation` モデルで管理する。

```python
class Corporation(BaseModel):
    corporation_name = models.CharField(max_length=255)
    corporation_type = models.CharField(max_length=100, blank=True)
    homepage_url = models.URLField(blank=True)

    has_care_management = models.BooleanField(default=False)
    has_home_nursing = models.BooleanField(default=False)
    has_home_care = models.BooleanField(default=False)
    has_day_service = models.BooleanField(default=False)
    has_day_care = models.BooleanField(default=False)
    has_geriatric_health_facility = models.BooleanField(default=False)
    has_senior_housing = models.BooleanField(default=False)
    has_paid_nursing_home = models.BooleanField(default=False)

    def __str__(self):
        return self.corporation_name
```

# 5. 法人サービス項目の考え方

Version 0.1 では、法人サービスは Boolean で管理する。

理由は以下の通り。

- 初期開発をシンプルにするため
- 営業分析ではまず有無が重要であるため
- 事業所名や住所の詳細管理は後から拡張可能であるため

将来的に必要があれば、法人サービスを別テーブル化する。

```text
Corporation
  1
  │
  多
CorporationService
```

# 11. 営業分析との関係

法人マスタは、営業可能性指数・競合度の計算に利用する。

## 11.1 営業可能性への影響

|項目|営業分析上の意味|
|---|---|
|法人内居宅あり|外部居宅への紹介余地が小さい可能性|
|法人内居宅なし|外部居宅への紹介余地が大きい可能性|
|法人内訪問看護あり|法人内完結の可能性|
|法人内老健あり|退院先が法人内施設に流れる可能性|
|サ高住あり|法人内入居導線あり|
|有料老人ホームあり|法人内入居導線あり|

# 12. 法人内完結度

法人内完結度は、法人が持つサービス数から算出する。

例：

```text
法人内完結度 =
居宅介護支援
+ 訪問看護
+ 訪問介護
+ 通所系サービス
+ 老健
+ 住まい系サービス
```

法人内完結度が高いほど、外部居宅への紹介可能性は低下する可能性がある。

ただし、実際には法人内で対応困難なケースや医療依存度の高いケースが外部へ紹介されることもあるため、営業履歴と合わせて判断する。

# 14. 設計上の注意点

- 法人情報と病院情報は分離する
- 初期段階では法人サービスはBooleanで管理する
- 事業所名や住所などの詳細情報はVersion1.0以降で検討する
- 法人内居宅あり＝営業不可ではない
- 法人内サービスの有無は公開情報と現場情報を照合する
- Web調査結果は備考欄に情報源を記録する
- 営業判断は法人情報だけでなく病院機能・営業履歴・紹介実績と合わせて行う
