# 開発思想

Hospital CRM は「営業支援システム」である。

単なる顧客管理（CRM）ではなく、

「病院・法人・地域連携・営業活動」を可視化し、
営業担当者の意思決定を支援することを目的とする。

本システムは

「情報量 × 鮮度 × 速度」

をコンセプトとし、

経験だけに依存しない営業を実現する。

# 設計原則

本システムは以下の原則に従って設計する。

- シンプルに作る（Keep It Simple）
- 後から拡張できる設計とする
- データを一元管理する
- 同じ情報を二重管理しない
- AIが利用しやすいデータ構造にする

# 1. 基本方針

本システムは、居宅介護支援事業所の病院営業活動を支援するための業務システムである。

開発では以下を重視する。

- 小さく作る
- 後から拡張できる
- データを資産として扱う
- 現場で入力しやすい
- 将来Django化・AI分析へ発展できる

# 3. データベース方針

初期段階では SQLite を採用する。

理由は以下の通り。

- 事業所単体での利用を想定する
- 同時アクセス数が少ない
- バックアップが容易
- Django標準で扱いやすい
- 将来PostgreSQLへ移行可能

# 5. 命名規則

アプリ名は英語・小文字・複数形とする。

例：

- hospitals

- contacts
- visits

## 5.2 モデル名

モデル名は単数形・パスカルケースとする。

例：

```python

Corporation
ContactPerson
VisitRecord

```

---

## 5.3 フィールド名

フィールド名は英語・小文字・スネークケースとする。

```python
hospital_name

community_beds
recovery_beds
chronic_beds
msw_count
```

主要モデルには以下の共通フィールドを持たせる。

# 7. ID設計

基本は Django 標準の `BigAutoField` を使用する。

ただし、外部共有やURL表示が必要なデータには UUID の利用を検討する。

---

# 8. 外部キー方針

病院と法人は以下の関係とする。

```text
Corporation 1 --- 多 Hospital
```

営業履歴・担当者・紹介実績は Hospital に紐づける。

```text
Hospital 1 --- 多 VisitRecord
Hospital 1 --- 多 ContactPerson
Hospital 1 --- 多 ReferralRecord
```

---

# 9. コメント方針

コードコメントは、処理の説明よりも「なぜそうしているか」を記載する。

悪い例：

```python
# 病院を取得する
hospital = Hospital.objects.get(id=hospital_id)
```

良い例：

```python
# 営業履歴は病院単位で蓄積するため、先に病院を取得する
hospital = Hospital.objects.get(id=hospital_id)
```

---

# 10. マイグレーション方針

- モデル変更後は必ず `makemigrations` を実行する
- マイグレーションファイルはGit管理する
- 本番運用後は既存フィールド名の変更を慎重に行う
- 削除よりも `is_active=False` を優先する

---

# 11. Git運用方針

コミットメッセージは以下の形式を基本とする。

```text
種別: 内容
```

例：

```text
feat: 病院マスタモデルを追加
fix: 病床数インポート処理を修正
docs: 開発規約を追加
refactor: 営業スコア計算処理を整理
```

---

# 12. バックアップ方針

SQLiteファイルを定期的にバックアップする。

対象ファイル：

```text
db.sqlite3
```

バックアップ例：

```text
backup/db_20260703.sqlite3
```

---

# 13. 個人情報・機密情報の扱い

本システムでは、利用者個人情報は原則として扱わない。

扱う情報は以下を中心とする。

- 病院情報
- 法人情報
- 担当者情報
- 営業履歴
- 紹介件数

利用者名・診断名・詳細な相談内容は入力しない。

---

# 14. AI利用方針

AIは以下の用途に限定して利用する。

- 営業コメント生成
- 面談記録の要約
- 営業優先度の説明文生成
- 弱シグナル分析
- 次回訪問提案

AIの出力は最終判断ではなく、営業担当者の判断補助とする。
